# Session Logs

Every new project must keep this folder. It may initially contain only this
README, but the first meaningful planning, implementation, review, QA, release,
or durable-decision session must add a real dated log.

Save meaningful session change logs here as Markdown using dated filenames that
include the acting assistant, so Codex, Claude, and Gemini logs never collide:

`YYYY-MM-DD-short-topic-<actor>.md`   (actor = `codex`, `claude`, or `gemini`; `both` only for a genuinely joint log)

Each log should include date, actor/tool, user request, decisions, files changed,
commands or verification, remaining risks, next steps, and whether anything was
promoted to `/Users/kangsungbae/Documents/지식저장소`. Set the `Actor:` field inside
to match the filename. Append to your own actor's file when continuing the same
topic on the same day; do not overwrite another actor's log.

The project-local log is the original session evidence. The shared knowledge
store is only for promoted reusable summaries, not a replacement for this local
log.

Keep logs concise. Do not paste full chat transcripts or full tool logs.
