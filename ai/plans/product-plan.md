## 1. 핵심 게임 콘셉트

화면 중앙에 **지구**. 운석·소행성·혜성·행성 파편·보스 행성이 화면 바깥에서 등장.
직선이 아니라 지구 중력에 빨려들 듯 **나선형/소용돌이 궤도**로 접근. 플레이어는
손가락으로 화면을 그어 천체를 벤다.

핵심 감정:
> "지구에 닿기 직전까지 끌어들였다가 정확히 베어내는 긴장감."

빨리 베는 게임 아님. 좋은 플레이어는 판단한다:
- 어떤 운석이 먼저 지구에 닿는가
- 어떤 오브젝트는 베면 안 되는가
- 지구 가까이 기다렸다가 고득점을 노릴 것인가
- Solar Lance 쏠 때 구조대/에너지 캡슐을 안 맞출 수 있는가
- 스킬을 지금 쓸 것인가, 보스까지 아낄 것인가

## 2. 핵심 디자인 원칙

### 2.1 지구 중심성
모든 핵심 시스템은 지구를 중심으로 돈다. 적은 지구로 접근, 플레이어는 지구를 방어,
스킬은 지구 기준 발동, 점수는 지구와의 거리·방어 성공·생존 시간으로 결정.

### 2.2 입력은 단순, 판단은 깊게
기본 조작은 긋기 하나. 내부 판단: 베야 하는가 / 아군 피해야 하는가 / 어느 방향 /
스킬 각도인가 / Last Save 노릴까.

### 2.3 즉시 판정, 지연 연출 — **Instant Judgment, Delayed Effect**
> 손을 뗀 순간 판정 즉시 확정, 이후 이펙트는 시각 연출.

Solar Lance는 손 뗀 순간 직선 경로 위 대상만 판정. 레이저 연출 중 구조대가 지나가도
추가로 안 맞음. 억울함 방지. 플레이어가 느껴야 할 것: "내가 맞췄다 / 내가 실수해서
구조대 맞췄다 / 게임이 나를 억까하지 않았다."

## 4. 메인 게임 화면 구조

### 4.1 기본 화면
- 화면 중앙: 지구
- 지구 주변: 방어막 / Earth Energy 링
- 바깥: 적 등장 영역
- 지구 주변 위험 구역: Last Save 판정 구역
- 상단: 점수, 생존 시간, Threat Gauge
- 하단/지구 주변: 스킬 게이지 링
- 화면 전체: 슬래시 입력 가능 영역

### 4.2 기본 좌표 개념
```text
C = (earthX, earthY)
R = earthRadius
```

| 구역 | 거리 기준 | 의미 |
|------|-----------|------|
| 외곽 궤도 | 4.0R 이상 | 안전 구역 |
| 중간 궤도 | 3.0R~4.0R | 주의 구역 |
| 위험 궤도 | 2.0R~3.0R | 충돌 임박 |
| Last Save 구역 | 1.3R~2.0R | 지구 직전 방어 |
| 충돌 구역 | 1.2R 이하 | 지구 피격 |

## 5. 핵심 게임 루프

### 5.1 한 판 흐름
1. 게임 시작 → 2. 지구 중앙 표시 → 3. 적 천체 바깥 등장 →
4. 소용돌이 궤도로 접근 → 5. 손가락으로 베어 파괴 →
6. 적이 지구에 닿으면 Earth Energy 감소 → 7. 0이면 게임 종료 →
8. 생존 시간·점수·보스 단계·콤보 산출 → 9. 랭킹전이면 서버 제출.

### 5.2 핵심 반복 재미
```text
다가온다 → 판단한다 → 벤다 → 콤보 터진다 → 더 어려워진다 → 스킬 쓴다 → 더 오래 버틴다
```

## 9. 스킬 상세

### 9.1 Orbital Cut
지구 주변 원형으로 그으면 궤도 칼날이 한 바퀴 돌며 위험 구역 적을 벤다.
- 발동: 지구 포함 원형, 시작/끝점 가까움, 각도 변화 충분, 완벽한 원 아니어도 됨.
- 판정: 시작-끝 거리 ≤ 전체 경로 25% / 원 안에 지구 중심 포함 / 회전 270도+ /
  최소 지구 반지름 1.5배 / 최대 화면 짧은 축 70%.
- 효과: 위험 궤도 내 강한 데미지, 작은 운석 즉파, 큰 소행성 큰 피해, 보스 제한 피해.
  인공위성·구조대·캡슐 맞으면 패널티.

### 9.2 Solar Lance
지구 관통 긴 직선 → 그 방향 강력 태양광 레이저.
- 발동: 지구 바깥 시작 → 중심/근처 통과 → 반대편 바깥까지, 손 뗀 순간 판정 확정.
- **최종 판정 원칙: 손 뗀 순간 직선 경로 위 대상만 판정. 이후 레이저 연출은 시각
  효과이며 연출 중 지나간 구조대/캡슐은 추가로 안 맞음.**
- 판정: 최소 길이 화면 짧은 축 60%+ / 직선성 0.88+ / 선분-지구중심 거리 ≤ 0.6R /
  시작·끝 1.5R 바깥 / 게이지 80+ / 쿨타임 12초.
  `직선성 = 시작-끝 거리 / 실제 경로 총 길이` (1.0 근접 = 직선).
- 효과: 경로 따라 레이저, 작은·일반 운석 즉파, 큰 소행성 큰 피해, 금속 운석에 방향
  무시 피해, 보스 약점 강한 피해.
- 패널티: Defense Satellite(-에너지, 콤보 끊김) / Energy Capsule(회복 상실) /
  Rescue Shuttle(Friendly Fire, 큰 패널티, 레이저 중단) / EMP Mine(화면 흔들림) /
  구조대(레이저 그 지점에서 중단).
- 규칙: 구조대가 경로에 있으면 큰 패널티. 단 연출 중 뒤늦게 지나가면 안 맞음.

### 9.3 Gravity Slow
지구 주변 돼지꼬리/소용돌이 → 중력장 왜곡, 적 감속.
- 발동: 스킬 링에서 시작, 0.2~0.3초 누른 후 작게 말아 들어감, 회전 180도+.
- 판정: 시작 1.2R~2.5R / 입력 0.3~1.2초 / 회전 180도+ / 중심은 지구 근처 /
  효과 2.5초 / 속도 50% 감소.
- 효과: 모든 적 2.5초간 50% 감속, 보스 25~35%만, 구조대 거의 안 느려짐,
  Last Save 쉬워짐.

### 9.4 Delta Shield
지구 포함 삼각형 → 지구 주변 삼각 방어막 (생존기).
- 발동: 지구가 삼각형 내부, 3개 큰 꺾임, 시작/끝 가까움, 너무 작은 삼각형 불인정.
- 판정: 꼭짓점 3개 전후 / 시작-끝 거리 ≤ 전체 경로 30% / 삼각형 내부 지구 중심 포함 /
  최소 지구 반지름 2배 / 지속 3초 / 쿨타임 30초.
- 효과: 3초 방어막, 작은 운석 닿으면 파괴, 큰 소행성 1회 막고 내구 감소, 보스 1회/
  피해 감소, 구조대·에너지 캡슐은 통과. 랭킹전 마지막 몇 초 버티기용.

## 12. Last Save 시스템

### 12.1 개념
지구에 거의 닿기 직전 적을 베면 발생. 표시 `LAST SAVE!` / 한국어 `지구 직전 방어!`.

### 12.2 거리 배율
| 구역 | 배율 |
|------|--:|
| 외곽 궤도 | x1.0 |
| 중간 궤도 | x1.4 |
| 위험 궤도 | x2.2 |
| Last Save | x3.5 |

### 12.3 보상
점수 x3.5, 게이지 +8, 콤보 유지, 특별 이펙트, 랭킹 보조 지표 기록.

### 12.4 밸런스 운영 기준
모니터링: 시도율 / 성공률 / 실패 후 사망률 / 전체 점수 중 비중.
목표:
```text
전체 점수 중 Last Save 비중: 15~30%
상위권 점수 중 비중: 25~40%
성공률: 40~60%
```

| Master | 극한 | 20~45초 | 60초+면 고수 |

Master는 60초 이상 버티기 어려운 수준 목표.

### 14.2 정렬 기준
1 생존 시간 → 2 총 점수 → 3 보스 처치 수 → 4 남은 Earth Energy →
5 최대 콤보 → 6 Last Save 횟수 → 7 정확도 → 8 기록 달성 시각.
유저 화면엔 과다 정보 노출 금지.

### 14.3 랭킹전 광고 원칙
랭킹전 도중 부활 **금지**. 허용: 실패 → 광고 → 1회 재도전권 → 새 판.
금지: 도중 사망 → 광고 → 그 자리 부활 → 기록 이어감.
> 광고는 기록을 이어주는 것이 아니라 새로 도전할 기회를 주는 것.

## 15. 난이도와 웨이브

### 15.1 적 이동 (나선형 접근)
```text
angle += angularSpeed * dt
radius -= approachSpeed * dt
x = earthX + cos(angle) * radius
y = earthY + sin(angle) * radius
```
적별 변수: 시작 반경 / 접근 속도 / 회전 속도 / 크기 / 체력 / 피해량 / 속성 /
분열 여부 / 방향 베기 필요 여부.

### 15.2 난이도 상승 (숫자만 늘리지 않음)
접근 속도, 회전 속도, 적 밀도, 금지 오브젝트, 방향 베기, 보스 개입, 충돌 반경 증가.

### 15.3 지구 충돌 반경 난이도 (**Gravity Swell**)
이미지 키우기보다 충돌/중력장 반경을 키움.
| 난이도 | 충돌 반경 |
|--------|--:|
| Rookie | 1.00배 |
| Defender | 1.08배 |
| Elite | 1.16배 |
| Master | 1.25배 |

## 23. 기술 구현 방향

### 23.1 렌더링
WebView 실행 → DOM 애니메이션 지양. 권장 Canvas/WebGL (PixiJS/Phaser).
비추천: React state로 운석 100개 이동, DOM 요소 게임 렌더링, CSS 애니메이션 남발.

### 23.2 권장 구조
```text
/src
  /game
    GameApp.ts  GameScene.ts  Earth.ts  OrbitSpawner.ts  ObjectManager.ts
    GestureSystem.ts  SkillSystem.ts  CollisionSystem.ts  ScoringSystem.ts
    RankingSystem.ts  BossSystem.ts  WaveGenerator.ts  RemoteConfig.ts  Telemetry.ts
  /data
    enemies.json  skills.json  bosses.json  difficulty.json  scoring.json
  /ui
    StartScreen.tsx  ResultScreen.tsx  RankingScreen.tsx  ModeSelectScreen.tsx
```

### 23.3 성능 원칙
60fps 목표, 저사양 최소 30fps. Object Pooling 필수. Sprite Atlas. 파티클 제한.
무거운 필터 최소화. 입력 판정 가볍게. 충돌은 원/선분 기반 단순화.

### 23.4 게임 루프
`requestAnimationFrame` + `deltaTime` → 위치 업데이트 → 충돌 → 입력 → 렌더.

### 23.5 Remote Config (앱 업데이트 없이 조정)
운석 점수/피해/등장률, Last Save 배율·판정 반경, 스킬 쿨타임·게이지 획득량,
